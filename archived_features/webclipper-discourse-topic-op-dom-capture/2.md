# Discourse 论坛帖子的 URL 归一化与抓取优化

`https://linux.do/t/topic/1867095` 对于这样一个帖子，如果最后再加上 `/20` 的后缀，就表示是当前这个帖子的xx条回复。而 `/1` 表示的就是帖子的主内容。

而我们当前的这个右侧评论侧边栏，会因为最后这个后缀的变化，导致切换到新的url中。

并且仍然是这个后缀，导致会fetch同样一个帖子为不同的文章——我们要做的是，只要是同一个帖子topic，就fetch贴主的内容，而对于其他人的回复，就配合右侧侧边栏进行抓取即可。

**需求：Discourse 论坛帖子的 URL 归一化与抓取优化**

**背景：**

Discourse 论坛（如 [linux.do](http://linux.do/)）的帖子 URL 结构为 `/t/slug/topic-id`，访问不同楼层时会变成 `/t/slug/topic-id/N`。当前 WebClipper 把带不同后缀的 URL 当作不同文章处理。

**问题：**

1. 同一帖子的不同楼层 URL 被识别为不同文章，产生重复的 article conversation
2. 评论面板（inpage-comments-panel）跟着 URL 变化跳转，丢失上下文
3. 用户在非首楼页面触发抓取时，可能抓到的不是楼主原帖内容

**期望行为：**

1. **URL 归一化**：所有 `/t/slug/topic-id/N` 归一到 `/t/slug/topic-id`，作为唯一 canonical URL
2. **文章抓取**：始终只抓取楼主发的第一条消息（OP），不抓回复内容
3. **评论面板稳定性**：用户在同一帖子的不同楼层间跳转时，评论面板保持同一个会话，不重置
4. **回复互动**：用户可以手动选中页面上的回复文字，通过「选中引用 + 评论」的方式进行互动
5. **适用范围**：通用 Discourse 检测，不限于 linux.do 单站
