# WebClipper：其他 AI chats 中通过 `$` 提及并插入本地 item（首版 ChatGPT + NotionAI）

## 背景 / 触发

- 用户希望在“其他 AI chats”的输入框里，获得接近 Notion AI `@` 的提及体验，但触发符使用 `$`。
- 当前 WebClipper 已有“复制为 Markdown 文本”能力与本地 item 数据源；但缺少“在站点输入框内选 item 并直接插入”的交互链路。
- 需求在讨论中已明确 pivot：不是做跳转式 deep-link，也不是做外链动作，而是做输入框内实时候选 + 插入。

## 核心需求（原始需求精炼）

1. 在 ChatGPT、NotionAI 输入框中，用户输入单个 `$` 后立即弹出小型候选窗口。
2. 候选窗口打开后，用户继续输入文本时要实时过滤。
3. 候选范围为本地全部 item（`chat + article`）。
4. 过滤字段固定为：标题、来源、URL 域名；当仅输入 `$` 时按最近保存时间倒序展示。
5. 必须支持键盘导航：`↑/↓` 移动高亮，`Tab`/`Enter` 选中当前高亮并插入。
6. `Esc` 仅关闭候选窗口，不删除输入框中已有 `$关键词` 文本。
7. 选中候选后，必须“替换本次触发片段（如 `$abc`）”，并在当前光标位置插入。
8. 插入文本必须复用现有“复制为 Markdown 文本”真源（与当前 copy 行为一致）。
9. 本次明确不做截断：无论内容长度，直接插入完整文本。
10. 首版范围仅覆盖 ChatGPT 与 NotionAI，不提前扩展到其他站点。

## 默认值与兼容策略

- 默认行为：在 ChatGPT/NotionAI 中默认启用 `$` 提及能力；并提供 Settings 开关 `ai_chat_dollar_mention_enabled`（默认 `true`，用户可关闭）。
- 已有用户升级后默认同样启用；不涉及 DB schema 迁移。
- 当前代码基线已切换到“会话列表分页读取”；mention 候选检索必须走独立存储查询，不复用分页列表接口，避免漏掉未加载项。
- 不在支持站点（或找不到可编辑输入框）时静默不激活，不影响现有采集/保存流程。
- 未来扩展策略：通过 editor adapter 注册机制扩展其他站点，保持触发/过滤/插入状态机不变。
- 备份导入/导出：由于备份会导出绝大多数 `storage.local`（仅排除少量敏感 key），因此新开关会随备份一起被导入/导出（无需额外接线）。

## 非目标（明确不做什么）

- 不做全文检索（message 正文级搜索）。
- 不做自动发送消息（仅插入文本，不触发提交）。
- 不做多平台一次性全覆盖（仅 ChatGPT + NotionAI）。
- 不改“复制 Markdown 文本”格式协议本身。
- 不加入截断策略、分段插入策略或平台专属长度策略。

## 验收标准（可检查）

- 在 ChatGPT / NotionAI 输入框中输入 `$`，候选窗口会出现，并可随输入实时过滤。
- 过滤结果与排序满足：标题/来源/域名匹配；空查询按最近保存时间倒序。
- `↑/↓` 可切换高亮；`Tab/Enter` 插入高亮项；`Esc` 仅关闭窗口且保留文本。
- 选中后替换触发片段（`$query`）并在光标位置插入，不破坏输入框其余文本。
- 插入内容与“复制为 Markdown 文本”同源且一致（不截断）。
- 变更完成后至少通过：`npm --prefix webclipper run compile`、目标测试、`npm --prefix webclipper run build`。
