# Idea: 防盗链域名管理 UI

## 背景 / 触发

- 当前 `ANTI_HOTLINK_REFERER_MAP` 硬编码在 `image-download-proxy.ts` 中，用户无法自行管理防盗链域名。
- 用户希望把少数派、知乎等站点的 referer 规则放到 Settings 里自己维护。
- 这次实现路径选择了 Settings 门控，而不是 context menu：`General` 里已经有 `web_article_cache_images_enabled`，并且 Notion OAuth 的「高级选项」按钮模式可以直接复用。

## 核心需求

1. 在 `Settings → General` 的「自动保存网页文章图片到本地 Beta」下方增加一个 `[高级选项]` 按钮，展开后显示防盗链域名管理面板。
2. 管理面板支持列表编辑：新增、修改、删除、编辑后持久化、重置默认值。
3. 默认种子只提供 `cdnfile.sspai.com -> https://sspai.com/`；用户删除后不应自动复活。
4. 运行时下载图片和文章图片自动缓存必须读取同一份持久化配置，而不是继续依赖硬编码表。
5. 域名必须是 hostname，referer 必须是 http(s) URL；非法输入不能静默写入。
6. 分层不能倒置：`platform/*` 不得依赖 `services/*`；运行时查表能力应放在 platform 层，Settings 通过 services 封装调用。
7. 运行时读取配置失败时不能导致采集中断；需保留“可继续下载/采集”的安全回退行为。

## 默认值与兼容策略

- 首次缺失配置时，写入默认种子并作为初始值。
- 已存在的配置按用户保存结果使用，不再与代码内置表自动合并。
- 如果用户把列表清空，表示明确禁用该规则集合，不要回填默认值。
- 未来新增域名时，应当通过用户编辑保存进入本地配置，而不是再改硬编码表。
- 非法输入保留在 UI 草稿态并显式提示，不写入持久层。
- 新增 storage key 作为普通非敏感设置键，保持与现有备份导出/导入策略兼容。

## 非目标

- 不新增独立 Settings section。
- 不做 context menu 入口。
- 不支持 wildcard / path 级规则，只保留 hostname + referer 的最小模型。
- 不调整其他图片缓存开关的语义。

## 验收标准

- General 设置里有一个可展开/收起的高级面板。
- 新安装能看到默认的少数派规则。
- 用户可以增删改保存，刷新后仍然生效。
- 删除默认规则后不会自动回来，重置才会恢复默认值。
- 文章采集和图片下载对同一份配置同时生效。
- 输入非法 domain/referer 时 UI 有明确错误提示，且不会静默丢弃为“已保存”状态。
- 配置读取异常不会阻断文章采集主流程。
